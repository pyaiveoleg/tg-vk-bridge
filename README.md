# Мост Telegram ↔ VK

Фоновый бот, который синхронизирует личные сообщения:

- **TG → VK** — все входящие сообщения, адресованные тебе в Telegram (текст, фото,
  ссылки, голосовые, видео, файлы), пересылаются в диалог с твоим VK-сообществом.
- **VK → TG** — твои ответы в этом VK-диалоге уходят обратно нужному человеку в Telegram.

Хостинг — бесплатный 24/7 (Oracle Cloud Always Free). Платить не нужно.

---

## Как это устроено

| Сторона | Что используется | Почему |
|---|---|---|
| Telegram | **userbot** (Telethon, твой личный аккаунт через MTProto) | Обычный бот Telegram **не видит** чужие входящие сообщения. Чтобы пересылать то, что пишут **тебе**, нужен userbot. |
| VK | **бот-сообщество** (Bot Long Poll API) | Бесплатно, легально, не блокируется. Ты читаешь/отвечаешь в диалоге со своим сообществом. |

### Маршрутизация ответов
Все входящие из разных TG-чатов попадают в один VK-диалог. У каждого чата есть
короткий токен в заголовке, напр. `🔵 Алиса · #827c`.

- **Ответить конкретному человеку** → сделай **reply** в VK на его сообщение
  (токен берётся из цитаты).
- **Просто написать текст без reply** → ответ уйдёт в **последний активный** чат
  (удобно для быстрой переписки с тем, кто только что написал).

## Шаг 1. Telegram API
1. Зайди на <https://my.telegram.org> → **API development tools**.
2. Создай приложение, скопируй **api_id** и **api_hash**.
NB: может быть ошибка "ERROR" без объяснения причин. Это скорее всего из-за несовпадения региона ВПН и региона сим-карты. Точно работает заходить с российского IP-адреса. Остальные способы не пробовались.

## Шаг 2. VK сообщество
1. Создай (или возьми существующее) сообщество: <https://vk.com/groups> → «Создать сообщество».
2. **Управление** → **Настройки** → **Сообщения** → включи «Сообщения сообщества».
3. **Управление** → **Настройки** → **API** → **Long Poll API**:
   - включи, версия — последняя (5.199);
   - в **Типы событий** включи **«Входящее сообщение»** (`message_new`),
     **«Редактирование сообщения»** (`message_edit`) — для синка правок,
     **«Реакция на сообщение»** (`message_reaction_event`) — для синка реакций,
     а также события удаления, прочтения и набора текста (`message_delete`,
     `message_read`, `message_typing_state`), если они доступны в интерфейсе сообщества.
4. **Работа с API** → **Ключи доступа** → создай ключ со scope:
   **сообщения (messages)**, **фотографии (photos)**, **документы (docs)**. Это `VK_TOKEN`.
5. Узнай числовой id сообщества: открой `vk.com/club12345678` → число `12345678` это `VK_GROUP_ID`.
6. Узнай свой личный VK id: `vk.com/id12345` → `12345` это `VK_OWNER_ID`.
7. **Важно:** открой диалог со своим сообществом и напиши ему любое сообщение —
   иначе сообщество не имеет права писать тебе первым.

## Шаг 3. Конфиг и вход
```bash
cp .env.example .env
# впиши TG_API_ID, TG_API_HASH, VK_TOKEN, VK_GROUP_ID, VK_OWNER_ID

python3 -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt

python login.py     # введи телефон + код из Telegram, скопируй строку в TG_SESSION в .env
```

## Шаг 4. Локальный запуск (проверка)
```bash
python bridge.py
```
Напиши себе в Telegram с другого аккаунта — сообщение должно прийти в VK-диалог с сообществом.
Ответь reply в VK — придёт обратно в Telegram.

## Деплой на Oracle Cloud (бесплатно, 24/7)

1. Зарегистрируйся: <https://www.oracle.com/cloud/free/> (нужна карта для верификации,
   списаний нет). Создай **Always Free** инстанс: *Compute → Instances → Create*,
   shape **VM.Standard.A1.Flex** (ARM, Always Free), образ **Ubuntu 22.04**.
2. Зайди по SSH и поставь Docker:
   ```bash
   sudo apt update && sudo apt install -y docker.io docker-compose-plugin git
   sudo usermod -aG docker $USER && exec sg docker newgrp   # перелогинься при необходимости
   ```
3. Скопируй проект на сервер (`scp -r ./tg-vk-bridge ubuntu@<IP>:~` или `git clone`),
   и положи туда заполненный `.env` (с уже полученным `TG_SESSION`).
4. Запусти:
   ```bash
   cd tg-vk-bridge
   docker compose up -d --build
   docker compose logs -f        # смотреть логи
   ```
   `restart: unless-stopped` поднимет контейнер после перезагрузки сервера.

> `TG_SESSION` лучше получить **локально** (`python login.py`) и перенести строку в `.env` —
> на сервере нет интерактивного ввода кода.

### Альтернатива: без Docker (systemd)
```bash
cd ~/tg-vk-bridge
python3 -m venv .venv && .venv/bin/pip install -r requirements.txt
sudo cp systemd/tgvk-bridge.service /etc/systemd/system/
# поправь пути/User в файле, если отличаются
sudo systemctl daemon-reload
sudo systemctl enable --now tgvk-bridge
sudo journalctl -u tgvk-bridge -f
```

## Настройки (.env)

| Переменная | Описание |
|---|---|
| `TG_API_ID`, `TG_API_HASH` | из my.telegram.org |
| `TG_SESSION` | строка сессии userbot (из `login.py`) |
| `VK_TOKEN` | ключ доступа сообщества (scope: messages, photos, docs) |
| `VK_GROUP_ID` | числовой id сообщества |
| `VK_OWNER_ID` | твой личный VK id (адресат пересылки и фильтр отправителя) |
| `FORWARD_GROUPS` | `true` — пересылать ещё и групповые чаты/каналы TG (по умолчанию только ЛС) |
| `TG_WHITELIST` | Белый список диалогов (опционально). Если задан — обрабатываются **только** перечисленные чаты; если пусто — все. Через запятую/пробел, можно `id` и `@username`. Напр.: `TG_WHITELIST=123456789, @durov, -1001234567890` |
| `REACTION_POLL_INTERVAL` | раз в сколько секунд явно запрашивать реакции Telegram; `0` отключает опрос |
| `REACTION_POLL_MESSAGES` | сколько последних сообщений проверять на новые реакции |
| `SYNC_READS` | синхронизировать доступные платформам статусы прочтения |
| `READ_POLL_INTERVAL` | интервал резервной проверки read-state VK; `0` отключает polling |
| `SYNC_TYPING` | передавать статусы набора текста/записи |
| `VK_VIDEO_MAX_MB` | максимальный размер видео, скачиваемого по ссылке VK |
| `DATA_DIR` | где хранить `store.json` |

## Синхронизация правок
Редактирование текста синхронизируется в обе стороны: правка собеседника в Telegram
меняет соответствующее сообщение в VK (с пометкой «(ред.)»), а твоя правка в VK —
сообщение в Telegram.

- В настройках сообщества включи событие **«Редактирование сообщения»** (`message_edit`).
- Синхронизируется **текст/подпись**; смена вложений при редактировании не переносится.
- Работает только для сообщений, отправленных после добавления этой функции
  (нужна связь id из [store.py](store.py)).

## Синхронизация реакций
Реакции синхронизируются в обе стороны: реакция собеседника в Telegram появляется
в VK на соответствующем сообщении, а твоя реакция в VK — в Telegram.

- В настройках сообщества должно быть включено событие **«Реакция на сообщение»**
  (`message_reaction_event`), см. Шаг 2.3.
- VK не публикует таблицу соответствия id реакций и эмодзи, поэтому карта задана
  «на глаз» в [reactions.py](reactions.py). Если реакция отображается не тем эмодзи —
  поправь словарь: бот логирует неизвестные id (`VK reaction: неизвестный id=...`),
  по ним легко определить реальное соответствие.
- Кастомные премиум-эмодзи Telegram в реакциях не переносятся (берётся дефолт).
- Дополнительно мост явно опрашивает реакции у Telegram. Это обходит случай, когда
  MTProto присылает накопленные изменения только после открытия диалога в приложении.
- Опрос охватывает и входящие сообщения собеседника, и ответы, которые ты отправил
  из VK: чаще всего собеседник ставит реакцию именно на второй тип сообщений.
- Для неоткрытых чатов используется только `messages.getUnreadReactions`.
  Тяжёлый batch-запрос отключён, чтобы не получать Telegram `FloodWait`.
- Точный VK reaction id `36` (`💅`) принимается от пользователя, но отвергается
  методом `messages.sendReaction` для сообщества. В направлении TG → VK он
  заменяется ближайшей доступной реакцией `😍`; VK → TG остаётся точным `💅`.

## Удаления сообщений
Удаление сообщения Telegram удаляет соответствующее сообщение сообщества в VK.
Если удалён элемент альбома, удаляется всё VK-сообщение с этим альбомом.

В обратную сторону удаление работает, только если VK присылает сообществу событие
`message_delete`/`message_deleted`. Доступность этого события зависит от текущего
варианта Long Poll API и настроек сообщества.

## Медиагруппы и видеокружки
- Альбом Telegram отправляется в VK одним сообщением с несколькими вложениями.
- Несколько фото/видео из VK отправляются в Telegram как одна медиагруппа.
- Если VK отдаёт прямой файл видеосообщения, он отправляется в Telegram как
  видеокружок.
- Telegram → VK: API сообщений сообщества не предоставляет отдельного формата
  видеокружка, поэтому такой файл прикрепляется как обычное видео/документ.

## Прочтение и набор текста
- Когда ты читаешь пересланное сообщение в VK и VK отдаёт `message_read`, исходный
  диалог Telegram помечается прочитанным.
- Когда собеседник читает ответ в Telegram, соответствующий входящий диалог
  сообщества VK помечается прочитанным.
- Набор текста в Telegram отображается в VK как активность сообщества.
- Набор текста в VK отправляется в последний активный Telegram-чат. У VK typing
  не содержит reply-контекста, поэтому точнее определить адресата невозможно.
- VK различает меньше видов активности: запись голосового и видеокружка со стороны
  Telegram отображается в VK как запись аудиосообщения.

## Репосты и ссылки VK
Вложения типа `wall`, включая `copy_history`, разворачиваются в текст поста и его
фото/документы. Ссылки вида `vk.com/wall...` также раскрываются через `wall.getById`.
Обычные ссылки `vk.com/video...`, `vk.com/clip...` и `vkvideo.ru/video...` раскрываются
через `video.get`. Если VK не вернул прямой MP4, мост использует `yt-dlp` и embed URL
из поля `player`. Закрытое, удалённое или слишком большое видео остаётся ссылкой.
Лимит загрузки задаётся `VK_VIDEO_MAX_MB`.

## Возможные вопросы
- **«Сообщество не может писать первым»** → напиши ему один раз из своего VK (Шаг 2.7).
- **Видео из VK** приходит файлом, когда API возвращает прямой MP4 URL; иначе ссылкой.
- **Безопасность:** `.env`, `*.session`, `store.json` — секретные, не коммить (уже в `.gitignore`).
- userbot работает от твоего аккаунта; не злоупотребляй авто-рассылками, чтобы не словить
  ограничения Telegram.
